# Minimum Common Denominator (Open Aggregator)

## Definition

The Minimum Common Denominator (MCD) is the minimal invariant structural core that enables interoperability while remaining extensible.

## Properties

- Structural minimalism
- Shared constraint layer
- Open extension surface
- Deterministic governance
- Aggregation without fragmentation

## Formal Object

Let `S_i` represent individual system instances. Let `F(S_i)` denote the invariant structural features extracted from each system.

```text
MCD = ⋂_{i=1}^{n} F(S_i)
```

The core is contained in all systems. Extensions exist outside the invariant core.

## OPT-IN Cross-Reference

The OPT-IN five-axis topology (`O-P-T-I-N`) is the MCD for programme management across any IDEALE pillar.

## Interoperability Boundary

| Layer | Description |
|---|---|
| Invariant core | Shared structure present in every participating system instance. |
| Constraint layer | Deterministic rules required for governance and interoperability. |
| Extension surface | Domain-specific additions that remain outside the invariant core. |
| Aggregation surface | Open aggregator interface that combines systems without fragmenting the shared core. |
